# 면접 스케줄링 시스템 - 집단면접 지원 버전

## 개요

기존 개별(individual) 면접만 지원하던 시스템에 집단면접(parallel, batched) 모드를 추가한 확장 버전입니다.

## 주요 기능

### 1. 면접 모드

- **individual**: 1:1 개별 면접 (기존)
- **parallel**: 같은 공간에서 여러 명이 각자 다른 활동 수행
- **batched**: 같은 공간에서 여러 명이 함께 같은 활동 수행 (집단토론, 프레젠테이션 등)

### 2. 3단계 계층적 최적화

batched 활동이 있을 때 자동으로 3단계 계층적 최적화를 수행합니다:

1. **Level 1: 그룹 구성**
   - 지원자들을 그룹으로 구성
   - 직무별 분리 우선 옵션
   - 유사 직무 그룹핑 지원

2. **Level 2: Batched 활동 스케줄링**
   - 그룹 단위로 batched 활동 시간표 생성
   - 방 용량과 선후행 제약 고려

3. **Level 3: Individual/Parallel 활동 통합**
   - 고정된 batched 시간표에 individual/parallel 활동 추가
   - 개인별 완전한 면접 일정 생성

### 3. 백트래킹 전략

- 하위 레벨에서 해를 찾지 못하면 상위 레벨로 백트래킹
- 각 레벨별 시간 제한 설정
- 최대 3회 재시도

## 사용 방법

### 1. UI에서 설정

1. **면접 활동 정의**에서 모드 선택
   - individual, parallel, batched 중 선택
   - batched 모드는 min_cap ≥ 2 설정 필요

2. **집단면접 설정** (batched 활동이 있을 때만 표시)
   - 직무별 분리 우선 옵션
   - 유사 직무 그룹 설정

3. **운영일정 추정** 실행
   - batched 활동이 있으면 자동으로 계층적 솔버 사용

### 2. 그룹 크기 호환성

모든 batched 활동의 그룹 크기 범위가 겹쳐야 합니다:
- 예: 집단토론(4-6명), 프레젠테이션(3-5명) → 공통 범위 4-5명

**⚡ 신규 기능: 자동 조정 가이드**
- 호환성 오류 시 두 가지 조정 옵션 제시
  - 옵션 1: 가장 넓은 범위 사용
  - 옵션 2: 평균값 기준 조정
- 각 활동별 구체적인 변경 사항 표시

### 3. 그룹 구성 미리보기

**⚡ 신규 기능: 그룹 구성 시뮬레이션**
- 실제 스케줄링 전에 그룹 구성 결과 미리보기
- 시뮬레이션 버튼 클릭으로 즉시 확인
- 표시 정보:
  - 총 그룹 수
  - 평균 그룹 크기
  - 직무 혼합 그룹 수
  - 각 그룹의 상세 구성

### 4. 성능 모니터링

**⚡ 신규 기능: 레벨별 실행 시간 추적**
- 각 최적화 레벨별 소요 시간 측정
- 백트래킹 발생 횟수 카운트
- 로그에 성능 요약 정보 제공:
  ```
  === 최적화 완료 (총 소요시간: 15.3초) ===
  **레벨별 실행 시간 요약:**
    - Level 1: 2.1초
    - Level 2: 5.4초
    - Level 3: 7.8초
  **백트래킹 발생 횟수:** 0회
  ```

### 5. 결과 확인

- 그룹별로 묶여서 표시
- group_id 컬럼으로 그룹 구성 확인 가능
- **⚡ 신규: 그룹별 체류시간 통계**
  - 그룹 단위 체류시간 분석
  - 그룹 내 직무 구성 표시

## 테스트

```bash
# 계층적 솔버 테스트 실행
python test_hierarchical_solver.py
```

테스트 시나리오:
- basic: 개별 면접만
- mixed: individual + parallel
- batched: individual + batched
- complex: 모든 모드 혼합
- stress: 대규모 지원자

## 주의사항

1. **성능**
   - batched 활동이 많을수록 계산 시간 증가
   - 지원자 50명 이하: 1분 내
   - 지원자 100명 이하: 5분 내

2. **제약 사항**
   - 그룹 크기 호환성 필수
   - 직무별 분리는 선호사항 (강제 아님)
   - 백트래킹 시 제약 완화 가능

3. **에러 처리**
   - 그룹 크기 비호환 시 명확한 에러 메시지
   - 백트래킹 실패 시 상세 로그 제공
